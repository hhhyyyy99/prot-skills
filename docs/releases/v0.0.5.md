# Prot Skills v0.0.5

Release workflow overhaul, bulk skills sync, and app polish.

## Highlights

- Split release workflow into three focused skills (version, publish, tag) for clearer release steps
- Added blocking AI review workflow with evidence-based methodology
- Added bulk sync action with inline progress for my-skills

## Added

- AI review workflow in CI with retry logic, self-approve, and structured findings
- Bulk sync action for my-skills with inline progress indicator
- oxlint + oxc-formatter, commitlint, and lint-staged for consistent code quality
- Repository workflow rules (branching, coding, commits) in `.agents/rules/`

## Changed

- Release flow now uses branch-and-PR workflow instead of direct-on-main
- Migrated repo scripts to TypeScript
- Consolidated test files into `src/tests/`
- Applied oxc linter and formatter across the codebase
- Removed Gemini provider support from AI review

## Fixed

- Complete built-in tool icon coverage with official Kiro icon and OpenCode mapping
- Tightened workspace header spacing
- Releases now published publicly instead of draft
- CI AI review false positives reduced through tighter prompts and client-side truncation

## Known Issues

- macOS builds are not signed yet. If Gatekeeper blocks the app, remove the
  quarantine attribute:

  ```sh
  xattr -d com.apple.quarantine "/Applications/Prot Skills.app"
  ```

## Download Notes

- Download the asset that matches your platform from the release assets below.
- Windows and Linux packages may use different installer formats depending on
  the build target produced by Tauri.
