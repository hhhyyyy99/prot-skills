# Prot Skills v0.0.5

This release brings a blocking AI code review pipeline, a new release-ship
skill, stricter linting and formatting tooling, and a batch of CI reliability
fixes.

## Highlights

- Pull requests now run an AI-powered code review that can block merging on
  critical findings.
- A new `release-ship` skill automates the branch-and-PR workflow for tagged
  releases.
- The project now enforces oxlint, oxc-formatter, commitlint, and lint-staged
  on every commit.

## Added

- Blocking AI review GitHub Actions workflow with support for Anthropic and
  DeepSeek providers.
- `release-ship` skill for automated version bumps and PR creation.
- Bulk sync action on the My Skills page with inline progress feedback.
- Repository workflow rules in `.agents/rules/` for branching, coding, and
  commit conventions.

## Changed

- Repository scripts migrated from JavaScript to TypeScript.
- Test files consolidated into `src/tests/` for a flatter, more predictable
  layout.
- Gemini provider removed from the AI review pipeline.
- Workspace header spacing tightened for a more compact layout.

## Fixed

- AI review prompt rewritten to reduce false positives and require evidence for
  every finding.
- Network errors and 429 rate limits in the AI review pipeline now retry
  reliably.
- DeepSeek thinking-only responses and non-standard Anthropic API responses
  handled correctly.
- Tool icon coverage completed for all built-in tools.
- Chinese README updated to reflect the current Tauri 2.x stack.
- GitHub releases now publish publicly instead of as drafts.

## Known Issues

- macOS builds are not signed yet. If Gatekeeper blocks the app, remove the
  quarantine attribute:

  ```sh
  xattr -d com.apple.quarantine "/Applications/Prot Skills.app"
  ```
